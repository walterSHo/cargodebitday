# AGENT

## Start here
Перед задачей обязательно прочитать:
1. `docs/PROJECT_RULES.md`
2. `docs/CONTEXT.md`
3. `docs/TASKS.md`
4. `docs/CHANGELOG.md`

## How to work
- Сначала зафиксируй scope задачи (что меняем / что не трогаем).
- Найди минимально затрагиваемые файлы.
- Не делай лишний рефакторинг.
- Не меняй бизнес-логику без явного запроса.
- Делай маленькие итерации.
- Если задача большая — предложи разбивку шагами.

## Finish checklist
После изменений:
1. Обнови `docs/CHANGELOG.md`.
2. Если появились новые идеи/хвосты — обнови `docs/TASKS.md`.
3. Если изменились правила/договоренности — обнови `docs/CONTEXT.md` и/или `docs/PROJECT_RULES.md`.
4. Кратко зафиксируй, что изменено и как проверить.

## Forbidden without explicit approval
- Ломать текущие формулы расчетов.
- Радикально менять UI.
- Удалять существующие блоки.
- Массово переименовывать сущности.
- Делать большой “косметический” рефакторинг.

## Prompt preamble (copy-paste)
Перед началом работы прочитай `AGENT.md`, `docs/PROJECT_RULES.md`, `docs/CONTEXT.md`, `docs/TASKS.md`, `docs/CHANGELOG.md`. Работай точечно, не меняй бизнес-логику без прямого запроса, после изменений обнови CHANGELOG и при необходимости TASKS/CONTEXT.
