# PROJECT_RULES

## 1) Что это за проект
- Рабочий B2B-калькулятор в стиле Cargo Parts.
- Не лендинг и не маркетинговая страница.
- Включает 2 инструмента:
  - Калькулятор середнього відтермінування.
  - Калькулятор мотивації / ЗП.

## 2) Блоки интерфейса
- Середнє відтермінування.
- Місячна мотивація.
- Квартальний бонус.
- Підсумок / розшифровка.
- Експорт / копіювання.
- Дисклеймер.
- Підпис автора (если присутствует в текущей версии UI).

## 3) Ключевая бизнес-логика (фиксировать)
- CRM: только checkbox (100% / нет).
- Бонус за виконання плану: ступенчатый расчет.
- Если середня знижка >= 27.5%: уменьшается только бонус за виконання плану на 50%.
- Просрочка: 3 отдельные поля в EUR:
  - 1–10 днів
  - 11–20 днів
  - 20+ днів
- Штраф по просрочке: сначала в EUR, потом конвертация по курсу.
- Курс должен быть видим в интерфейсе.
- Місячна и квартальна логика разделены.
- Квартальные группы: ввод по 3 месяцам.
- Среднее по группе = (M1 + M2 + M3) / 3.
- Группа выполнена при average >= 95%.
- PROFIT / JOKER считается обычной группой.
- Бонус по количеству выполненных групп:
  - 7 = 12000
  - 8 = 16000
  - 9 = 22000
  - 10+ = 30000
  - все 12 = +10000 сверху.

## 4) UI/UX правила
- Строгий Cargo Parts / B2B portal.
- Без flashy SaaS-стиля и лишнего декора.
- Интерфейс компактный, плотный, аккуратный.
- Улучшения должны повышать usability, а не добавлять “красоту ради красоты”.
- Квартальный блок держать читаемым и по возможности плотным по высоте.
- Summary должно быть понятным и полезным.

## 5) Ограничения
- Не менять бизнес-логику без явного запроса.
- Не делать крупный рефакторинг без необходимости.
- Работать точечно.
- Учитывать, что `docs/index.html` используется для Pages-версии.

## 6) Правила для AI-агента
- Перед правками читать `docs/PROJECT_RULES.md` и `docs/CONTEXT.md`.
- Не менять логику без подтверждения.
- Не раздувать scope.
- Предпочитать маленькие итерации.
- Описывать изменения коротко и по делу.
